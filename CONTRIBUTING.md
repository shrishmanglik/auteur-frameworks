# Contributing

Contributions are welcome through pull requests.

## Before opening a PR

1. Open an issue describing the production problem.
2. Keep the change provider-neutral unless capability evidence is included.
3. Add or update typed schemas.
4. Add deterministic tests.
5. Run npm run check.
6. Confirm that npm run audit:public passes.

## Evidence standard

Every new framework or QC rule must state the failure or workflow it addresses, evidence class, scope, falsification condition, and synthetic test fixtures.

Do not submit private prompts, media, record identifiers, customer work, credentials, provider session data, or unpublished mappings.

## Review

Maintainers review correctness, scope, evidence quality, API stability, safety, and publication boundaries. A contribution may be declined even when the code works if the evidence or ownership is unclear.
